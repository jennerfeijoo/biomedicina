Desarrollo teórico

Los conceptos se estudian desde su significado, sus relaciones, la evidencia que los sustenta y los límites de interpretación.

Concepto 1

Pruebas

Definición. Procedimientos planificados que producen evidencia sobre cumplimiento, comportamiento o error respecto a criterios definidos.

Alcance. En Modelado y Simulación en Biomedicina, el tema de pruebas se estudia dentro de verificación y validación. Dominarlo exige responder qué magnitudes intervienen, cómo se relacionan y bajo qué supuestos; además de la definición hay que reconocer componentes, relaciones, condiciones de aplicación y señales de error.

Relaciones. Este concepto se interpreta junto con Datos de referencia y Incertidumbre. La conexión debe expresarse como una cadena explicable: qué información entra, qué transformación o mecanismo ocurre, qué resultado se observa y qué evidencia permitiría comprobarlo. La ruta recomendada es: definir variables y unidades → elegir una representación → derivar o ejecutar el método → comprobar el resultado → interpretar su alcance.

Aplicación. En el caso de separación de error numérico y error de modelo, el concepto ayuda a formular una pregunta precisa y a seleccionar observaciones o mediciones pertinentes. La conclusión debe apoyarse en coherencia dimensional, derivación, cálculo reproducible y contraste con casos límite. Debe recordarse que un resultado numérico puede ser correcto y, aun así, responder a un modelo inadecuado.

Comprobación de dominio

  • Definir pruebas con palabras propias y delimitar qué queda fuera del concepto.
  • Representar su relación con Datos de referencia y Incertidumbre mediante un esquema, tabla, modelo o procedimiento.
  • Proponer una evidencia que apoyaría la interpretación y otra observación que la debilitaría.
  • Identificar al menos un supuesto, una fuente de incertidumbre y un caso límite.

Caso biomédico resuelto

Situación. Un equipo académico necesita estudiar separación de error numérico y error de modelo. Debe integrar Pruebas, Datos de referencia y Incertidumbre y producir una recomendación verificable sin usar datos personales ni intervenir en pacientes.

  1. Pregunta. Determinar cómo verificación y validación puede aportar a separación de error numérico y error de modelo, sin convertir un resultado educativo en una decisión clínica.
  2. Representación. Organizar Pruebas, Datos de referencia y Incertidumbre en un mapa que muestre entradas, relaciones, resultados y supuestos.
  3. Método. Seguir la secuencia de razonamiento cuantitativo: definir variables y unidades; elegir una representación; derivar o ejecutar el método; comprobar el resultado; interpretar su alcance.
  4. Evidencia. Solicitar coherencia dimensional, derivación, cálculo reproducible y contraste con casos límite; registrar procedencia, unidades, parámetros y criterios de aceptación.
  5. Conclusión. Redactar por separado lo observado, lo inferido y lo que todavía requiere verificación. La propuesta queda limitada al uso educativo y al contexto descrito.
Ver criterio de solución

Una solución sólida entrega un modelo, cálculo o simulación comentada; declara la pregunta, los datos o evidencias necesarios, el procedimiento, los criterios de calidad, el resultado y sus límites. No afirma causalidad, eficacia o seguridad más allá de la evidencia disponible y explica cómo se comprobaría cada afirmación.

Actividad guiada

Producto: Modelo, cálculo o simulación comentada sobre separación de error numérico y error de modelo.

  1. Delimita. Formula una pregunta, el contexto de uso y el resultado esperado. Incluye Pruebas, Datos de referencia y Incertidumbre.
  2. Prepara. Usa un caso simulado, literatura o datos abiertos sin identificadores personales. Registra fuente, licencia, versión y diccionario de variables.
  3. Resuelve. Aplica esta secuencia: definir variables y unidades → elegir una representación → derivar o ejecutar el método → comprobar el resultado → interpretar su alcance. Conserva cálculos, decisiones y resultados intermedios.
  4. Verifica. Contrasta con coherencia dimensional, derivación, cálculo reproducible y contraste con casos límite; incluye un control, un caso límite o una comparación alternativa.
  5. Comunica. Entrega una página con método, resultado, interpretación, incertidumbre, limitaciones y siguiente prueba necesaria.

Autoevaluación con respuestas

1. ¿Qué significa dominar pruebas en esta unidad?

Respuesta esperada: Poder definirlo, delimitarlo, relacionarlo con Datos de referencia y Incertidumbre, aplicarlo a un caso y reconocer supuestos y límites.

2. ¿Cómo se relacionan pruebas y datos de referencia?

Respuesta esperada: La respuesta debe mostrar dirección y mecanismo o procedimiento: qué entra, qué cambia, qué sale y qué evidencia permite verificar la relación.

3. ¿Qué secuencia de trabajo evita saltar directamente a una conclusión?

Respuesta esperada: Definir variables y unidades → Elegir una representación → Derivar o ejecutar el método → Comprobar el resultado → Interpretar su alcance.

4. ¿Qué evidencia sería pertinente para separación de error numérico y error de modelo?

Respuesta esperada: Coherencia dimensional, derivación, cálculo reproducible y contraste con casos límite, elegida de acuerdo con la pregunta, el contexto y el tipo de afirmación.

5. ¿Cuál es el principal límite de interpretación?

Respuesta esperada: Un resultado numérico puede ser correcto y, aun así, responder a un modelo inadecuado.

6. ¿Qué debe contener una entrega reproducible?

Respuesta esperada: Pregunta, fuente y versión de datos, supuestos, procedimiento, parámetros, resultados intermedios, comprobaciones, conclusión y limitaciones.

Glosario operativo

  • Pruebasconcepto

    Procedimientos planificados que producen evidencia sobre cumplimiento, comportamiento o error respecto a criterios definidos.

  • Datos de referenciaconcepto

    Representaciones registradas de observaciones o mediciones cuyo significado depende de contexto, metadatos y proceso de obtención.

  • Incertidumbreconcepto

    Conocimiento incompleto sobre el valor de una medición, parámetro, modelo o conclusión, que debe expresarse y propagarse.

  • Supuestorazonamiento

    Condición aceptada para construir un modelo o argumento y que debe declararse para poder evaluar su alcance.

  • Evidenciamétodo

    Observación, medición o resultado documentado que apoya o contradice una afirmación.

  • Incertidumbrecalidad

    Grado de conocimiento incompleto sobre una medición, parámetro, modelo o conclusión.

  • Validacióncalidad

    Proceso para comprobar que el resultado es adecuado para el uso previsto dentro de razonamiento cuantitativo.
